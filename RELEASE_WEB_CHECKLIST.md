## 1. 本版完成范围

- 25 名基础哲学家、四大阵营、角色技能、羁绊、哲人王和王城屏障。
- 十波三路防守、W5/W9 压力点、Boss 与确定性遭遇变化。

- V7 对局存档、V3 局外档案、手动存档、导出/导入 JSON 和导入前回滚。
- 胜负结算、最终阵容、阵营/羁绊和伤害/承伤/治疗/护盾四榜。
- 桌面和手机共用同一引擎、规则与浏览器存档。
- 自动入口 `/`，强制桌面 `/desktop`、`/?ui=desktop`，强制手机 `/mobile`、`/?ui=mobile`。

- 横屏手机战场、地图下沿备战棋子、独立五卡商店、点选部署/交换/撤回/出售。
- PWA manifest、正式图标、standalone metadata、更新检查和保守离线启动页。
- 音乐/音效分轨、静音、音量持久化和正式 WAV 接口。
- `/api/health` 与 `/api/release` 公开只读接口。

## 4. 存档与兼容

- 当前对局存档版本 V7；未来版本存档会安全拒绝，不会用空档覆盖。
- V4/V5/V6 迁移测试通过；战斗临时状态不会写入稳定存档。
- 历史随机保存 `seed + cursor`，W3/W6 候选、重试、重新加载和波次检查点可重放。
- `paidCost`、免费棋子、自由主义退款、免费刷新和战争机器奖励均有幂等测试。
- 局外档案 V3 独立保存；未知未来内容 ID 不能进入当前运行时角色池。
- 导入文件先完整迁移校验，再一次性写入；失败不覆盖，成功导入前保留可恢复备份。
- 浏览器数据被清理仍会丢失，玩家应使用“导出存档文件”留存跨设备备份。

## 5. 手机与 PWA

- 手机横屏是主要游玩方向；竖屏提示可关闭且同一会话不重复。
- 不强制原生方向锁；浏览器允许时尝试请求横屏，失败时仍可手动旋转继续。
- 使用 `100dvh`、safe-area inset、局部滚动和内部可滚动弹窗。
- 页面没有主要全局横向溢出；桌面拖放与手机点按操作共存。
- PWA 不缓存游戏 HTML/RSC/JS，只缓存离线说明页和图标。
- **真机验收尚未完成**：至少还需一台 Android Chrome 和一台 iPhone Safari。
- 部署后还需验证 HTTPS 下的添加到主屏幕、standalone 启动、安全区、动态地址栏、后台切回和 Service Worker 新版本接管。

## 6. 难度与平衡结论

- 经济矩阵中优先升级明显强于只存息、前期死刷和随手消费；固定只买一个阵营或运营失误会被惩罚。
- 历史矩阵总体通关 75/783；快速升级 68/261，存息 1/261，前期死刷 6/261。
- W5 是首个明显淘汰点，W6 回落，W9 是主压力峰；脚本 AI 不会像真人一样转阵、换位和止损，不能把模拟胜率直接当真人胜率。
- 五月风暴完成率 23.70%，世界大战 1.85%；激进主义 19.75%，保守主义 5.56%。这是需要真人重点挑战的设计风险，不在上线前凭启发式脚本再次改数值。
- 推荐真人测试顺序：五月风暴 + 激进主义、世界大战 + 保守主义、纯希腊、希腊混搭、德国快速成型、法国和英国终局。

## 7. 性能、资源与安全

- 当前 `dist/client` 约 **8.43 MiB**。
- 最大资源：旧地图 `battlefield-showcase.png` 约 2.43 MiB，正式地图约 2.27 MiB；旧地图仍作为兼容资源保留，是后续可清理的发布体积债务。
- CSS 约 0.30 MiB，主游戏 JS 约 0.25 MiB；25 张角色 WebP 和 WAV 音效均为本地静态资源。
- Next 配置级安全头在 Vinext Worker 实测中未生效，已撤回无效配置；`X-Content-Type-Options`、拒绝嵌入、referrer 和权限策略必须在正式托管层补齐并以 HTTPS 响应实测为准。
- 开发工具只在开发环境、localhost 且显式 `?devtools=1` 时开放。
- 仓库扫描未发现硬编码 API key、client secret、private key 或 password 赋值。
- 构建仍有三类已知非阻塞提示：代理环境变量、Vinext 动态路由分类提示、插件耗时提示。

## 9. 部署与回滚方案

1. 停止并行改动，形成唯一候选提交 SHA。
2. 对该 SHA 重跑 `npm.cmd test`、两套模拟和静音系统 Chrome。
3. 推送同一 SHA 到 GitHub；站点也必须从该精确源状态保存版本。
4. 先验证 `/api/health`、`/api/release`、`/`、`/desktop`、`/mobile` 和 PWA 更新。
5. 再开放公开访问。
6. 若发现 P0/P1：把站点回滚到上一已知良好版本；不改写玩家 localStorage。修复版继续支持 V7/V3，禁止降低存档版本或用空档覆盖。
