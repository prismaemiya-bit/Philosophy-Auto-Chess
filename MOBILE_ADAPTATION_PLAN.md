# 往哲荣耀 v0.2 手机适配方案

## 目标与边界

手机适配只改变界面组织、触控方式和可读尺寸，不改变战斗坐标、敌人路线、棋子数值、存档结构或桌面布局。手机以横屏为唯一游戏方向：竖屏显示全屏方向门禁并禁止误操作，横置后才进入可购买、部署、开波和查看信息的战术界面。

## 已落地的第一阶段

- 声明标准移动端 viewport，并保留浏览器缩放能力。
- 新增最后加载的 `mobile.css`，统一接管 999px 以下的历史响应式规则。
- 手机竖屏显示独立的横屏门禁；“进入横屏模式”会尝试请求全屏与浏览器方向锁定，不支持该 API 时提示玩家关闭系统竖屏锁并手动横置。
- 横屏保持 16:9 的真实战术画布，避免把二十个部署点和 Boss 强行缩成不可点击的小点。
- 保留现有“先点棋子，再点高亮格”流程作为触控部署主路径，拖放只是桌面增强。
- 备战区和商店改成横向滑动；卡面不缩字，操作按钮保持至少 44–48px 高。
- 羁绊和战斗情报改成横向摘要，详情、设置、历史抉择和棋子检查器限制在安全区域内并允许内部滚动。
- 使用 `safe-area-inset-*` 兼容刘海、圆角屏和底部手势条。

## 验收矩阵

| 视口 | 主要目标 |
| --- | --- |
| 360×800 竖屏 | 显示横屏门禁；底层游戏不可误触；提示关闭系统竖屏锁 |
| 390×844 竖屏 | 主门禁验收尺寸；全屏方向页和“进入横屏模式”按钮完整可见 |
| 844×390 横屏 | 完成购买、备战区点选、点击部署、开波、暂停和设置开关 |
| 768×1024 平板 | 战场全宽显示；侧栏转为横向摘要；卡面不降到不可读字号 |
| 1366×768 / 1920×1080 | 桌面坐标、备战区、商店三按钮和既有截图布局不得回退 |

## 第二阶段

1. 在系统 Chrome 中加入 390×844、844×390 和 768×1024 的 DOM 几何验收与截图。
2. 验证触控点击部署、横向滑动不误触购买、商店免费刷新时三按钮尺寸不变。
3. 验证 W3/W6 历史抉择、设置和结算在虚拟键盘与安全区下仍可滚动到最后一个按钮。
4. 对低端手机执行 2× 战斗性能采样；若帧率不足，只降低粒子表现，不降低规则 tick 或隐藏战斗信息。
5. 实机验收 iOS Safari 与 Android Chrome 的手动旋转、全屏请求和方向锁定降级提示；Web API 无权限时不得卡死在无说明状态。

## 发布门槛

- 页面本身没有横向滚动，只有战场、商店、备战区等明确的局部滑动区。
- 所有关键触控目标最小 44px，主要文字不依赖 hover 才能看见。
- 竖屏稳定阻断游戏交互；横屏能从新局走到结算；桌面系统 Chrome 回归通过。
- 不以自动化截图代替至少一台 iOS 和一台 Android 的真人触控验收。
