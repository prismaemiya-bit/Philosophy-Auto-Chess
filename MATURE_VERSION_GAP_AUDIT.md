# 《往哲荣耀》成熟版本差距审计

日期：2026-07-23  
基线：历史事件生产闭环、确定性遭遇、五名 Boss、任务档案 V3、局内存档 V7、声音设置 V2

## 当前已经达到的版本边界

- 十波核心循环、购买/部署/撤回/出售/三合一、战斗结算与失败重试已有自动回归和系统 Chrome 验收。
- W3 历史事件、宗教改革奖励、W6 意识形态和改良/自由/共产经济效果均有真实玩家入口，不再依赖测试辅助函数跨门禁。
- 历史事件随机流、工业免费刷新、战争机器奖励、公共供给与任务奖励具备保存/重载和幂等边界。
- 敌人具有可理解的职责名称与图标；W5/W10 共五名 Boss，外形、名称和阶段机制可区分，遭遇选择可按存档种子重放。
- 胜利页已追加本局历史、战争机器、经济、关键棋子与新任务的次级总结；系统 Chrome 已真实完成 W10 并验收该覆盖层。
- 基础 Web Audio 短音、音乐/音效分轨设置、六场景音乐素材接口、SSR/自动播放静默降级和快速操作节流已完成。
- 当前审计范围内没有已知 P0 或会阻断 W3/W6 的 P1。此结论不等于整个游戏不存在未知缺陷。

## 成熟公开版本前仍需完成的 P1

1. **正式敌人和 Boss 美术**：当前代码绘制徽记已可读且不丑化界面，但仍不是统一画风的正式美术包。应由正式素材替换，不使用临时 AI 图冒充成品。
2. **真人整局平衡数据**：以真实商店牌序连续完成多局，分别记录五名 Boss 的到达率、胜率、战斗时长、阵亡和失败原因；在此之前不同时改经济与 Boss 数值。
3. **输入和无障碍验收**：键盘全流程、焦点顺序、屏幕阅读器名称、色弱区分、减少动态效果、触屏命中和控制器方案尚未形成完整矩阵。
4. **存档长期产品化**：V1—V7 程序化迁移与 V7 压力矩阵已通过；仍需收集各发布构建的真实外部 fixture，并补未来历史子状态逐版本迁移链、多存档槽、导出/备份/损坏恢复。
5. **性能与兼容矩阵**：低端设备、长时间连续战斗、Chrome/Edge/Firefox 与不同缩放/分辨率需要发布前实机验收。
6. **声音成品**：基础合成短音、音乐/音效素材注册接口、分轨音量和静音已交付；正式授权音乐/音效文件、设备试听、动态范围校准和听觉无障碍仍未完成。

## 可延期的 P2 内容广度

- 新地图、额外模式、真正的角色/地图/Boss 任务奖励、局外成长、完整设置页和多语言。
- 更多历史事件与意识形态不应先于现有五名 Boss 真人平衡和存档迁移 fixture。
- 更大规模的战斗/UI 模块拆分只有在出现明确维护或性能证据时再做，当前不需要重写稳定核心。

## 发布边界

- 本阶段完成的是可继续试玩和迭代的成熟纵切，不宣称内容完整的 1.0。
- 运行页眉、`package.json` 与 `release-info.json` 仍保留上一正式版本 `0.1.1`；界面内“本版新增”标识 V0.2 候选内容。正式发布获准时必须统一升级版本元数据、发布说明、工件名与升级烟测，当前不借候选收口伪装成已发布 V0.2。
- 本轮未打包、未创建或发布 Release、未推送 GitHub；发布工件与升级烟测不属于本轮已验证范围。
