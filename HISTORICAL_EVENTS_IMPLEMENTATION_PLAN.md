# 历史事件系统实施契约

更新时间：2026-07-22

本文件是历史事件系统的续作入口。规则来源是用户本轮确认稿；如本文件与当前代码冲突，以当前代码、自动测试和本文件“已完成基础”一节为准。所有暂定数值必须集中配置、可测试、可调整，不能散落在 UI 或战斗循环中。

## 当前基线

- 开始本轮前：97/97 自动测试通过；TypeScript、工具脚本类型检查、ESLint、production build 全部通过。
- 工作区包含上一批主界面、任务、音频、经济模拟和 UI 修改，均为待保留用户工作；不得 reset、checkout 或覆盖。
- 对局保持单人、手动开波、十波、五张商店、25 名基础棋子、四大阵营和六小羁绊。
- W5 洞穴之影、W10 绝对精神和世界之夜不得被历史事件替换。
- 不实现法国革命街垒，不增加角色、地图、装备、新货币、初始金币、商店锁定、额外波数或移动版。

## 已完成基础，不得重做

### 独立确定性领域模型

`app/game/historical-events.ts` 已提供：

- 7 个稳定事件 ID 与 5 个稳定立场 ID；
- 数据驱动的兼容池；
- 独立 `seed + cursor` 随机流，不在抽取时调用 `Math.random`；
- W2 成功结算后生成并保存实际 W3 事件；
- W5 成功结算后生成并保存实际 W6 三个候选及顺序；
- 事件展示/完成、立场选择、奖励幂等、持续效果、波次覆盖、战争机器奖励和逐波经济标志的稳定字段；
- `pendingHistoricalDecision`，供 UI 与 `startWave` 在接入弹层后统一阻止跳过决定；
- 旧存档清洗与实际候选保留；
- 宗教改革排除保守/激进，资本积累排除共产主义。

事件随机种子用于历史事件、立场及事件明确赠送的确定性商店操作（工业免费刷新、改良替换），不与普通付费刷新或战斗随机流合并。更新事件池后，已保存的 `eventId` 和 `stanceCandidateIds` 必须继续作为权威结果。

### V7 对局存档

`app/game/engine.ts` 已将 `SAVE_VERSION` 升为 7，并把 `historicalEvents` 纳入：

- `GameState`；
- 稳定序列化；
- V6→V7 迁移；
- 未来版本拒绝。

旧存档迁移规则已经实现：

- 已越过 W3 的旧局不补事件；
- 已越过 W6 的旧局不补立场；
- 尚未越过触发点的旧局可正常触发；
- 旧局缺少事件种子时，从稳定存档内容派生，不使用临时随机数。

### 检查点和重试

`startWave` 已把历史事件状态深拷贝进 `WaveCheckpoint`。`retryWave`、`restartCurrentWave` 和战斗中稳定存档均恢复对应快照。因此重试同一波会重放相同事件流，而不会保存战斗临时状态或重新抽取结果。

### 实际购入成本

`Piece.paidCost` 已成为 V7 稳定字段：

- 正常购买记录实际卡费；
- 三合一累加三个参与棋子的有效成本；
- V6 棋子按现有卡费与星级推导成本；
- 后续宗教改革免费棋子必须写 `paidCost: 0`；
- 自由主义完整退款必须使用 `paidCost`，不能使用卡面满价；
- 普通出售仍保持当前 `saleRefund`，不要提前改变。

## 推荐代码边界

1. `historical-events.ts`：稳定 ID、注册表、配置、确定性随机、纯状态转换、兼容关系。
2. `engine.ts`：准备阶段动作、部署上限、购买来源、刷新/替换/出售、存档迁移。
3. `combat-core.ts`：需要冻结的事件战斗规则快照；不得从 React 状态读取规则。
4. `battle.ts`：开波应用、敌人/Boss、波次覆盖、结算收入与幂等奖励。
5. `GameClient.tsx`：只展示状态并调用引擎动作；不得自行结算奖励或决定随机结果。
6. `scripts/economy-sim.ts`、`scripts/balance-sim.ts`：真实规则模拟和可复制报告。
7. `tests/game-engine.test.ts`：确定性、迁移、效果、经济、Boss 与重试回归。
8. `tests/rendered-html.test.mjs`：最低限度渲染合同。
9. `tests/browser-interactions.mjs`：最终系统 Chrome 交互验收；不得使用 Codex 内置浏览器。

## 剩余批次

- 新增引擎动作：进入历史、选择宗教改革奖励、选择意识形态；内部旧函数名为兼容保留。
- W3 有待处理事件时禁止开波；W6 有候选但未选立场时禁止开波。
- 宗教改革：三个不同大阵营、当前可用的 2 费棋子；选择后 `paidCost: 0`；备战区满时保留待领取状态。
- 城邦危机：开波冻结 A/B/C 每路最靠前且基础阻挡为 1 的地面棋子；普通版有效阻挡 2、攻速代价暂测 10%；高阻挡棋子不得增强。
- 法国大革命：1—2 费攻击/最大生命暂测 +15%，4—5 费暂测 -10%，3 费不变；不改技能等级、羁绊人数、卡费或出售。
- 五月风暴普通版：部署上限 +1，可到 9；四大阵营最高档失效，小羁绊不变。
- 保守主义必须同时取消持续收益和代价；一次性奖励不追回。
- 改良主义：每波一次、同费用替换、使用事件随机流、不触发购买和资源机制。
- 五月风暴激进版：人口 +2，可到 10；四大阵营有效人数封顶 4，小羁绊按真实不同角色数计算。
- 保守主义取消超人口后不得删除/出售棋子；超人口时禁止开波并返回需撤回数量。

### 批次 C：经济事件与立场

- 先运行当前 `npm.cmd run economy:sim` 并保存摘要，再调数值。
- 使用“规则解析器”返回当前基础收入、利息上限、刷新成本等有效规则；不要运行时修改 `ECONOMY_RULES` 常量。
- 工业革命：正常购买支出计入 `waveFlags.normalPurchaseSpend`；免费/奖励/复制/派生不计；出售不回退；每波最多一次免费刷新且不结转。
- 资本积累：首测基础收入 -1、利息上限 +2；计算何时落后、追平、净收益和十波回本。
- 自由主义：每波一次，正常购买棋子按 `paidCost` 完整退款；免费棋子为 0；合成后累加；不得重复结算。
- 共产主义：无利息，保留基础收入，首测固定公共供给 2；资本积累局不得出现。
- 为所有可激进事件建立独立配置，不使用统一倍率。未给出的激进数值必须由模拟决定并记录为“待调”，不能猜成冻结规则。
